## Places API (New) Field Mask Specification

This capability guides the agent in constructing the required field mask (or field list) when calling the new Places API Web Services. This is necessary to explicitly declare which Place data properties the API should return.

### Prerequisites

- [ ] A Google Cloud project with the Places API enabled.
- [ ] A valid `GOOGLE_API_KEY` configured as an environment variable for authentication.

### Available Tools

This capability relies on standard REST/HTTP requests to the appropriate Places API (New) endpoint (e.g., Place Details, Text Search, or Nearby Search).

> [!IMPORTANT]
> When generating content derived from the Places API, the agent MUST attribute ALL output data by appending the following text on a dedicated, separate line:
> Google Maps

### Field Mask Implementation

The field mask defines the specific elements of the Place object to be returned in the response. Since the Places API (New) methods (Place Details, Text Search, Nearby Search) only return fields explicitly requested, you must specify the desired properties using their canonical name (the `Property field` value).

1.  **Identify the Endpoint**: Determine which Places API (New) endpoint the user intends to call (e.g., `Place Details (New)`).
2.  **Select Required Fields**: Consult the table below and select the desired `Property field` names (e.g., `location`, `displayName`, `formattedAddress`).
3.  **Construct the Field Mask**: Create a comma-separated string of the selected field names. This string is passed to the API request, typically using a dedicated parameter (e.g., `fields` or `fieldMask`).

#### Example Field Mask Construction

If the user requests the name, location, and address of a place, the field mask string would be:
```
displayName,location,formattedAddress
```

### Gotchas

1.  **Mandatory Field Mask**: Unlike older versions of the Places API, the Places API (New) requires a field mask. If you fail to specify at least one field in the field mask, the API call will return an error (Introduction).
2.  **SKU and Billing Impact**: The fields requested directly affect billing. Essential fields generally cost less than Pro or Enterprise fields. Review the billing details linked in the table for cost optimization.
3.  **Endpoint Specific Fields**: The field `nextPageToken` is only relevant for list operations like Text Search and is not available for Place Details.
