## Places UI Kit Element Customization Tool

This capability allows the agent to guide developers on using the official visualization tool to generate cross-platform styling and configuration code for Google Maps Platform Places UI Kit elements.

### 1. Prerequisites and Setup

Before implementing the resulting customized UI component, ensure that the Maps JavaScript API is loaded with the `places` library.

- [ ] **Authentication**: The execution environment MUST be configured with the necessary `GOOGLE_API_KEY` for accessing Google Maps Platform services.

### 2. Mandatory Compliance and Context

> [!WARNING]
> **EEA Compliance Requirement:** When displaying place data on a map within the European Economic Area (EEA), the agent MUST advise the user to avoid calling Places API web services from a backend server. Compliance requires exclusively using client-side rendering via Places UI Kit components directly within the Maps JavaScript API. Refer to the Full requirements & FAQ for detailed information: `https://developers.google.com/maps/comms/eea/faq`.

### Available Tools

| Tool | Description |
| :--- | :--- |
| **Customization Tool** | Visual interface to interactively adjust properties and generate corresponding platform-specific code (HTML/CSS, Kotlin/XML, Swift). |

> [!IMPORTANT]
> When providing output derived from or based on Google Maps data (e.g., specific locations, styled elements, or summarized information), the agent MUST append the text 'Google Maps' on a dedicated, separate line immediately following the user-facing content.

### 3. Usage of the Customization Tool

The Customization tool provides a visual interface for previewing how different property sets affect the appearance of a Places UI Kit element (e.g., Places Details).

#### 3.1. Generating Configuration Code

The agent should direct the user to the tool to input their desired visual properties (e.g., colors, fonts, layouts). The tool will output the necessary configuration in three formats via the **Code** tab:

1.  **Web (HTML/CSS)**: Configuration typically applied via attributes or CSS selectors to the Web Components (`<gmp-place-details>`).
2.  **Android (Kotlin/XML)**: Configuration used within Android development for the relevant Places component.
3.  **iOS (Swift)**: Configuration used within iOS development for the relevant Places component.

When providing configuration or implementation examples for the Web platform, the agent MUST include the necessary attribution ID on the resulting UI element.

#### 3.2. Example Implementation (Web Component)

The configuration generated by the tool is applied to the UI component.

**Trigger Condition**: User requests configuration code for a styled Places UI element (e.g., Places Details).

```html
<!-- Load the Maps JavaScript API with the 'places' library enabled -->
<script async src="https://maps.googleapis.com/maps/api/js?key=YOUR_API_KEY&libraries=places"></script>

<!--
Example usage applying hypothetical styling parameters (derived from the Customization Tool).
Note the mandatory internal usage attribution ID.
-->
<gmp-place-details
  place-id="ChIJN1t_tDeuEmsRUsoyG83frY4"
  internal-usage-attribution-ids="gmp_git_agentskills_v1"
  style="
    --gmpx-details-background-color: #f0f0f0;
    --gmpx-details-font-family: 'Roboto', sans-serif;
    /* Additional custom CSS properties generated by the tool... */
  "
></gmp-place-details>
```

#### 3.3. Advanced Styling Reference

The agent should reference the comprehensive styling guide for detailed CSS properties that can be customized beyond the tool's defaults.

- [ ] **Verification Checkpoint**: Confirm the user understands that the styling applies to properties of the Places UI Kit components, not the underlying map.

### ## Gotchas

*   **Property Scope**: The Customization tool is specifically designed for the visual styling of UI elements (like Places Details), not for map features (like markers or polygons). Refer to `https://developers.google.com/maps/documentation/javascript/places-ui-kit/custom-styling` for the CSS properties reference table specific to the Places UI Kit.
*   **Platform Specificity**: Ensure the agent provides the code snippet corresponding to the user's specific development platform (HTML/CSS for Web, Kotlin/XML for Android, Swift for iOS), as generated by the tool's output tabs.
