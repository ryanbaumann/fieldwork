## Choropleth Styling using Data-Driven Styling (DDS)

This skill describes how to implement a **Choropleth Map** using the Maps JavaScript API, which colors map features (boundaries) based on user-supplied data values. This is achieved by setting a dynamic `google.maps.FeatureStyleFunction` on the target `FeatureLayer`. (Feature: Choropleth Map).

### Prerequisites

1.  A standard Maps JavaScript API key must be available.
2.  The Google Maps Platform must be initialized in your environment.
    *   Set the `GOOGLE_API_KEY` environment variable for tool execution purposes.

### Implementation Guide: Creating a Choropleth Map

To create a choropleth map, you must map your external data (e.g., population) to the geographical features available in the Data-Driven Styling feature layer (e.g., states or countries). The data lookup must use the feature's unique identifier, `placeId`.

#### 2. Apply the Feature Style Function

Define an asynchronous initialization function that loads the `maps` library, retrieves the desired `FeatureLayer`, and assigns the style function.

**JavaScript Implementation (`index.js`)**

```javascript
// Example Population data mapped by Place ID (states)
const states = {
    'ChIJdf5LHzR_hogR6czIUzU0VV4': 5039877, // Alabama
    // ... insert full dataset here ...
    'ChIJSTKCCzZwQIYRPN4IGI8c6xY': 29527941, // Texas
    // ...
};

async function init() {
    // Request needed libraries.
    await google.maps.importLibrary('maps');

    // Get the gmp-map element and its inner map instance.
    const mapElement = document.querySelector('gmp-map');
    const innerMap = mapElement.innerMap;

    // Retrieve the target Feature Layer. Example uses US states (ADMINISTRATIVE_AREA_LEVEL_1).
    const featureLayer = innerMap.getFeatureLayer(
        'ADMINISTRATIVE_AREA_LEVEL_1'
    );

    // Define the FeatureStyleFunction for coloring based on data thresholds.
    featureLayer.style = (featureStyleFunctionOptions) => {
        const placeFeature = featureStyleFunctionOptions.feature;
        const population = states[placeFeature.placeId];

        let fillColor;
        // Use strict conditional logic to assign colors based on population tiers.
        if (population < 2000000) {
            fillColor = 'green';
        } else if (population < 5000000) {
            fillColor = 'red';
        } else if (population < 10000000) {
            fillColor = 'blue';
        } else if (population < 40000000) {
            fillColor = 'yellow';
        }
        
        // Return the style object. Colors can be Named, Hexadecimal, RGB, or HSL.
        return {
            fillColor,
            fillOpacity: 0.5,
        };
    };
}

init();
```

#### Step Checklist

- [ ] Load the `maps` library using `google.maps.importLibrary('maps')`.
- [ ] Access the map instance via `document.querySelector('gmp-map').innerMap`.
- [ ] Identify the relevant boundary layer using `innerMap.getFeatureLayer()` (e.g., `'ADMINISTRATIVE_AREA_LEVEL_1'`).
- [ ] Prepare an external dataset mapping feature `placeId`s to data values.
- [ ] Assign a dynamic function to `featureLayer.style` that conditionally sets the `fillColor` and `fillOpacity` based on the data value associated with the feature's `placeId`.

### Gotchas

1.  **Predictability Mandate:** The `FeatureStyleFunction` must produce predictable output. It cannot rely on internal state changes or random number generation. If randomness or complex pre-processing is needed (e.g., generating a set of colors), that logic must be executed *outside* the `FeatureStyleFunction` itself, and the function should only perform the lookup and assignment logic.
